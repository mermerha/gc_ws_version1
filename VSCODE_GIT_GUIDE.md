# VS Code 界面化 Git 操作指南

通过 VS Code 自带的“源代码管理 (Source Control)”插件，你可以完全不写命令行，只用鼠标点击就能完成代码的提交和上传。

## 基本流程 (日常代码更新)

### 1. 打开源代码管理视图
在 VS Code 最左侧的活动栏中，点击带有**分支图标**的按钮（通常是第三个，名为“源代码管理 / Source Control”），或者使用快捷键 `Ctrl+Shift+G`。
*当你修改了文件但还没有保存或提交时，这个图标上会显示一个数字，代表修改的文件数量。*

### 2. 暂存更改 (相当于 `git add`)
在源代码管理面板中，你会看到一个名为**“更改 (Changes)”**的列表，里面列出了所有被修改、新增或删除的文件。
- **暂存单个文件**：鼠标悬停在某个文件上，点击右侧出现的 `+` 号 (暂存更改 / Stage Changes) 按钮。
- **暂存所有文件**：鼠标悬停在“更改 (Changes)”这个标题行上，点击右侧的 `+` 号 (暂存所有更改) 按钮。
*暂存后，这些文件会移动到上面的**“暂存的更改 (Staged Changes)”**列表中。*

### 3. 提交更改 (相当于 `git commit`)
在面板顶部的文本框中（显示“消息 (Message)”的地方）：
1. 输入一段简短的文字来描述你这次修改了什么内容（例如："修改了无人机起飞参数" 或 "新增颜色识别节点"）。
2. 点击文本框下方的 **“提交 (Commit)”** 按钮（或者按 `Ctrl+Enter`）。

### 4. 推送到 GitHub (相当于 `git push`)
提交完成后，你的修改已经安全保存在本地。接下来需要上传到 GitHub 网页：
- 点击面板上的 **“同步更改 (Sync Changes)”** 按钮。
- 或者点击面板右上角的 `...` (更多操作) 菜单 -> 选择 **“推送 (Push)”**。
*等待几秒钟，VS Code 界面上方的进度条或左下角状态栏的转圈结束后，代码就成功传到 GitHub 了！*

## 常用进阶操作

### 查看差异 (Diff)
想看看自己到底改了哪几行代码？
- 直接在“更改”列表中**双击**某个文件。
- VS Code 会打开一个左右分屏（或上下分屏）的视图：左侧带有红色的是被删除的旧代码，右侧带有绿色的是新增的新代码。

### 撤销修改 (放弃更改)
如果你对某个文件改错了，想要恢复到上一次提交的干净状态：
- 在“更改 (Changes)”列表中，鼠标悬停在该文件上。
- 点击右侧出现的**弯曲向左的箭头图标**（放弃更改 / Discard Changes）。
- **注意**：这个操作是不可逆的，一旦放弃，你还没提交的修改将永久丢失！

### 拉取最新代码 (Pull)
如果你的队友在 GitHub 上更新了代码，或者你在另一台电脑上修改了代码，你需要下载最新版到当前电脑：
- 点击源代码管理面板右上角的 `...` -> 选择 **“拉取 (Pull)”**。
- 或者直接点击 **“同步更改 (Sync Changes)”**，它会自动先执行拉取 (Pull)，再执行推送 (Push)。

---
**💡 口诀总结**：修改代码 -> 去侧边栏点 `+` 号暂存 -> 输入一句话说明并点 `提交` -> 点击 `同步更改`。
